### MODIFY — templates
- `templates/project/main.ts.ejs.t` — add:
  ```ts
  if (config.openapi?.enabled) {
    const registry = app.get(OpenApiRegistry);
    const document = registry.build({
      title: config.openapi.title,
      version: config.openapi.version,
      description: config.openapi.description,
    });
    // register BearerAuth security scheme
    document.components = { ...document.components, securitySchemes: { bearer: { type: 'http', scheme: 'bearer', bearerFormat: 'JWT' } } };
    document.security = [{ bearer: [] }];
    SwaggerModule.setup(config.openapi.path, app, document);
  }
  ```

## Tests

1. **Smoke** — fresh-generated project boots, `GET /docs` returns HTML (Swagger UI), `GET /docs-json` returns full OpenAPI 3.0.3 doc with populated `components.schemas`, `paths`, and `security: [{ bearer: [] }]`.
2. **Config knob** — setting `openapi.path: /api-docs` moves the UI mount accordingly.
3. **Disabled mode** — `openapi.enabled: false` skips the entire SwaggerModule.setup block; no `/docs` route registered.
4. **BearerAuth interaction** — Swagger UI's "Authorize" button renders a Bearer token input; a request with `Authorization: Bearer <jwt>` reaches controllers with the token intact.
5. **OpenAPI-config subsystem install** — `bun codegen subsystem install openapi-config` injects the block cleanly.
