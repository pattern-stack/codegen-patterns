# SPEC-CLI-02: Entity Noun — Summary, New, List, Validate

**Status:** Draft
**Date:** 2026-04-13
**Depends on:** SPEC-CLI-01 (NounModule abstraction), ADR-015, ADR-016

---

## Purpose

Implement the `entity` noun — the most-used surface of the CLI. This spec covers:

- `codegen entity` — summary pane (entities defined, families, query counts) + dynamic hints
- `codegen entity new <yaml>` — generate one entity
- `codegen entity new --all` — batch-regenerate all entities
- `codegen entity list` — tabular list of defined entities
- `codegen entity validate [dir]` — schema + cross-reference validation

`codegen entity inspect <name>` is out of scope for this spec (deferred to CLI-04).

## Command Specifications

### `codegen entity`

Zero-verb summary. Generated by `buildNounSummaryCommand(entity)` from SPEC-CLI-01. No new command class needed.

**`summary(ctx)`** returns:

```typescript
{
  title: 'entities',
  body: [
    // If ctx.entityCount === 0:
    '  No entities defined yet.',
    '  Create one at entities/<name>.yaml to get started.',
    // Else: tabular list
    '  ▸ contact       synced      12 fields   3 queries',
    '  ▸ opportunity   synced       8 fields   2 queries',
    // ...
  ],
  footer: ctx.entityCount === 0
    ? undefined
    : `${count} entities · ${familyCount} families · ${queryCount} queries`,
}
```

**`hints(ctx)`** returns state-aware hints:

| State | Hints |
|-------|-------|
| Not initialized | `codegen init` — Initialize project |
| Initialized, no entities | `codegen entity new entities/example.yaml` — Generate first entity |
| Has entities, none generated | `codegen entity new --all` — Generate all |
| Has entities, some generated | `codegen entity new <file>`, `codegen entity validate` |
| Has validation issues | `codegen entity validate` — Review issues |

### `codegen entity new <yaml>`

Generate a single entity from a YAML definition.

**Class:** `EntityNewCommand`
**Path:** `[['entity', 'new']]`

**Options:**
- `yaml` — positional, required unless `--all` given. Path to `.yaml` file.
- `--all` — boolean flag. If set, regenerate all entities in `<configured entities dir>`. `yaml` positional is disallowed.
- `--dry-run` — boolean. Plan what would be written, do not write.
- `--force` — boolean. Skip git-safety check for uncommitted changes.
- `--only <target>` — `backend | frontend | clean-lite-ps`. Generate a subset.
- `--continue-on-error` — boolean. Used with `--all`. Report failures and continue.

**Behavior:**

1. Load context.
2. If `--all`: enumerate YAML files in `ctx.config.paths.entities` (default `entities/`).
3. For each target YAML:
   - Load + validate YAML via `loadEntityFromYaml`.
   - Resolve template set from config (`generate.backend`, `generate.frontend`, `generate.cleanLitePs`, filtered by `--only`).
   - Compute output paths. Collect existing files that would be overwritten.
   - Run `checkGitSafety(outputPaths)` unless `--force`. If any target has uncommitted changes, print warning with file list and require `--force` or interactive confirmation.
   - If `--dry-run`, print planned writes and return.
   - Invoke Hygen via `invokeHygen({ templateSet, locals })` from `src/cli/shared/hygen.ts`.
   - Report per-file outcome (written, unchanged, updated).
4. Print aggregate summary on success.

**Output (success, single entity):**

```
✓ validated opportunity.yaml
✓ resolved family=synced (base classes present)
✓ generated 14 files, 2 unchanged, 0 conflicts
→ src/opportunities/opportunity.entity.ts
→ src/opportunities/opportunity.repository.ts
  ...
```

**Output (`--all`):**

```
✓ contact         14 files, 2 unchanged
✓ opportunity     14 files, 2 unchanged
✗ account         validation failed — missing required field 'name'

5 entities · 4 succeeded · 1 failed
```

**JSON output:**

```json
{
  "command": "entity new",
  "entities": [
    {
      "name": "opportunity",
      "status": "success",
      "files": {
        "written": ["..."],
        "unchanged": ["..."],
        "skipped": []
      }
    }
  ],
  "totals": { "succeeded": 5, "failed": 0 }
}
```

**Exit codes:** 0 on all success, 1 if any entity fails.

### `codegen entity list`

Tabular list of entities. Shorter output than summary — pure data, no hints.

**Class:** `EntityListCommand`
**Path:** `[['entity', 'list']]`

**Options:**
- `--family <name>` — filter by family
- `--format <plain|tree|json>` — `plain` default, `tree` groups by family, `json` emits structured data

**Output (`plain`):**

```
NAME           FAMILY     FIELDS   QUERIES   TABLE
contact        synced     12       3         contacts
opportunity    synced     8        2         opportunities
activity       activity   6        0         activities
```

### `codegen entity validate [dir]`

Runs YAML schema + cross-reference validation. Exits non-zero on errors.

**Class:** `EntityValidateCommand`
**Path:** `[['entity', 'validate']]`

**Options:**
- `dir` — positional, optional. Defaults to `ctx.config.paths.entities`.
- `--strict` — treat warnings as errors.

**Behavior:** delegates to the existing `validateEntities()` in `src/index.ts`. Wraps output with `printSuccess` / `printWarning` / `printError` based on issue severity.

**Exit codes:** 0 if no errors (and no warnings in `--strict` mode), 1 otherwise.

## Shared Helpers

### `src/cli/shared/hygen.ts`

```typescript
export interface HygenInvocation {
  generator: string;      // e.g. 'entity'
  action: string;         // e.g. 'new'
  templateRoot: string;   // absolute path to templates/
  locals: Record<string, unknown>;
  cwd: string;
}

export async function invokeHygen(opts: HygenInvocation): Promise<HygenResult>;

export interface HygenResult {
  written: string[];
  unchanged: string[];
  errors: Array<{ file: string; message: string }>;
}
```

Wraps the existing Hygen invocation logic in `src/cli.ts`. Extracted verbatim — no behavior change. All nouns that generate files (entity, subsystem) share this helper.

### `src/cli/shared/git-safety.ts`

```typescript
export async function checkGitSafety(paths: string[]): Promise<{
  clean: boolean;
  dirty: string[];   // files with uncommitted changes
}>;
```

Runs `git status --porcelain -- <paths>` and returns which targets have uncommitted edits. Used by `entity new` and `subsystem install` to warn before overwriting. Non-git projects return `{ clean: true, dirty: [] }`.
